## 1. Project Setup

The project follows the same structure as project
[*se1-play*](https://github.com/sgra64/se1-play)
of previous assignments.

The project is built from git-modules: *.env*, *.vscode* and *libs*.
The *src* folder has sub-folders: *main* (Java source code), *tests*
(Unit tests) and *resources* (none-Java source code):

```sh
<workspaces>    # workspace folder within which project directories exist
 |
 +-<ordering-system>    # project folder of the project
   +--.gitignore        # patterns of files for git to ignore
   |
   +--.gitmodules       # file with information about git modules
   +-<.env>             # git-module with the sourcing script: 'env.sh'
   +-<.vscode>          # git-module with VSCode project settings and launch files
   +-<libs>             # git-module with libraries required by the project
   |  +--junit-platform-console-standalone-1.9.2.jar
   |  +-<junit>             # JUnit package for Java
   |  +-<jackson>           # package to process JSON data in Java
   |  +-<jacoco>            # code-coverage package for Java
   |  +-<logging>           # logging package for Java
   |  +-<lombok>            # code-injection package for Java
   |
   +-<src>              # <src> contains the project source code
   |  |
   |  +-<main>              # <src/main> Java source code (*.java files)
   |  |
   |  +-<tests>             # <src/tests> Unit tests
   |  |
   |  +-<resources>         # <src/resources> none-Java sources, properties files
   |     |
   |     +--application.properties  # application configuration
   |     +--log4j2.properties       # log4j2 logger configuration
   |     +-<META-INF>
   |        +--MANIFEST.MF  # packaging information for created .jar
   |
   +--<target>              # content generated by compilers and other tools
   |   |                    # including the final '.jar' artifact
   |   +--application-1.0.0-SNAPSHOT.jar
   |   +-<classes>          # compiled classes
   |   +-<test-classes>     # compiled test classes
   |   +-<resourses>        # content copied from 'src/resources'
   |   +-<docs>             # content created by javadocs compiler
   |   +-<src-delomboked>   # de-lomboked src
```

The initial project has two branches:

- Branch *"main"* with commits:

    ```
    8e69cad (HEAD -> main, tag: base) add src/main, src/tests, src/resources
    fb7f8e6 add .gitmodules
    02a8e65 add .gitignore                      <-- branch 'git-modules' here
    c5254ef (tag: root) root commit (empty)
    ```

- Branch *"git-modules"* branched off *"main"* at the second commit
    *"add .gitignore"*:

    ```
    42e9f32 (HEAD -> git-modules) add git-module 'libs'
    d925b8c add git-module '.vscode'
    edd09d4 add git-module '.env'
    02a8e65 (main) add .gitignore
    c5254ef (tag: root) root commit (empty)
    ```

The branch structure of the project should be:

```
commits:                                    branches
root  .gitignore .gitmodules   src

 (*) <-- (*) <----- (*) <----- (*)      <-- [main]
           \
            +- (*) <-- (*) <-- (*)      <-- [git-modules]

commits:      .env   .vscode   libs
```


Follow steps to setup the project:

1. Create a new project directory *"ordering-system"* in your workspace.

1. Create a local git repository with an empty root commit tagged as *"root"*
    on the *main* branch.

1. Install [*.gitignore*](.gitignore) and commit with message: *"add .gitignore"*.
    <!-- [*https://raw.githubusercontent.com/sgra64/ordering-system/refs/heads/main/.gitignore*](https://raw.githubusercontent.com/sgra64/ordering-system/refs/heads/main/.gitignore). -->

1. Create new branch *"git-modules"* and install *git-submodules* one after another
    as described in [*.gitmodules*](.gitmodules).
    Commit each module with message: `add git-module '(module-name)'`.

    Show the commit log on branch *git-modules* and compare with the log above.

1. Switch back to the *main* branch and checkout file *.gitmodules* from branch
    *.gitmodules*. Commit to the *main* branch with message: `add gitmodules`.

1. Fetch the initial *src* folder from:
    [*https://github.com/sgra64/ordering-system*](https://github.com/sgra64/ordering-system):

    - Set new remote *"se1-repo"* with the URL.

    - Fetch branch *main* from the remote *"se1-repo"*.

    - Checkout *src* from the fetched branch:
        ```sh
        git checkout se1-repo/main -- src       # checkout folder 'src' from fetched branch
        ```

    - Commit *"src"* folder to *main* branch with message:
        `add src/main, src/tests, src/resources`.

    - Tag commit with *"base"*.

1. Validate the project builds and tests run in both, the terminal and the IDE:

    - Source the project and start your IDE.

    - Perform a clean project build in the terminal:

        ```sh
        mk build        # perform a clean project build that should compile, run tests
                        # and create: 'target/application-1.0.0-SNAPSHOT.jar'
        ```

    - Run tests in the IDE and also in the terminal:

        ```sh
        mk run-tests
        ```
        ```
        ╷
        ├─ JUnit Jupiter ✔
        │  └─ Application_0_always_pass_Tests ✔
        │     ├─ test_001_always_pass() ✔
        │     └─ test_002_always_pass() ✔
        ├─ JUnit Vintage ✔
        └─ JUnit Platform Suite ✔
        ```

    - Run the program:

        ```sh
        mk run Hello World
        ```
        ```
        Hello, 'SE-1 Ordering System' (modular)
        - arg: Hello
        - arg: World
        ```

        Invoking the program without command line arguments outputs arguments
        taken from
        [*application.properties*](src/resources/application.properties):

        ```sh
        mk run
        ```
        
        ```
        Hello, 'SE-1 Ordering System' (modular)
        - arg: A
        - arg: BB
        - arg: CCC
        ```

1. When everything works, create a new remote repository named:
    *"ordering-system"* and push branch *main* to that remote repository.
